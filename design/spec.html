01

앱 개요 및 철학

OnStep이란?

OnStep은 뷰티·패션 라이프스타일을 체계적으로 관리하는 번들 자산 관리 & 지능형 루틴 시스템입니다. 복잡한 설정 없이 즉시 실행 가능한 Zero-Setting 원칙을 바탕으로, Setup이 설계한 데이터가 Today를 자동으로 구성합니다.

핵심 철학

①
Zero Setting
복잡한 설정 없이 즉시 실행. Setup에서 한 번 설계하면 Today가 자동으로 동작한다.
②
Bundle Asset Management
4개짜리·10ml 등 세트 단위의 Total/Current 수량을 정밀 추적. 잔량 관리 및 사용 기한 알림 체계화.
③
Intelligent Routing
날씨·시간대를 분석하여 오늘 실행해야 할 루틴과 스타일을 자동으로 제시한다.
④
Visual Archive
매일의 #OOTD/#MOTD가 달력에 스티커처럼 쌓여 라이프스타일 포트폴리오가 된다.

BOX — Inventory Engine V2

도메인 분류

도메인카테고리특화 필드
BEAUTYSkincare / MakeupTotal Qty, Current Qty, Open Date, Expiry Date
FASHIONOuter / Top / Bottom / Shoes / BagScene Tags (TPO), Material, Care Guide
ACCJewelry / Watch / Belt / Sunglasses / Hat / OtherMaterial Type, Special Note

Bundle Counter

Total vs Current 분리 추적. Segmented Bar로 시각화 (예: 4개 중 3개 남음 → 바 3/4 채움). Current가 Total의 25% 이하이면 경고 색상 표시 (Today 알림 연동은 추후 분석 예정).

제품 등록 Flow (3-Step)

1
분류 선택 — BEAUTY / FASHION / ACC
선택에 따라 하위 카테고리와 Step 3 특화 필드가 결정됨. UI: 세련된 실버 보더 버튼
2
공통 필드 (모든 분류 동일)
Object Photo (제품 단독), Storage Photo (보관 장소 — 저장 필수 조건 ★), Location Name, Brand, Product Name ★, Purchase Date
3
분류별 특화 필드
BEAUTY: Total Qty (+/-), Current Qty (+/-), Open Date, Expiry Date
FASHION: Scene Tags (#출근 #결혼식 #운동 등), Material, Care Guide
ACC: Material Type (금/은/가죽 등), Special Note

Validation 규칙

  • Storage Photo 미등록 시 저장 버튼 비활성화
  • Product Name 필수
  • Beauty: Current Qty ≤ Total Qty 자동 검증

BOX → Library 제품 연동

Log Library 탭에서 Makeup/Look 테마에 제품 추가 시 → onstep_box_assets에서 도메인별 검색 후 선택. 선택된 제품은 테마 카드에 칩으로 표시되며 Today에도 연동 노출.

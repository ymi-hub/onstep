### 4개 앱 분석 기반 기능·디자인 적용 로드맵

작성일: 2026-06-01  
분석 대상: Skincare Routine Diary · Skincare Routine App · 마이루틴 · Habit 앱군

## 1. 벤치마킹 통합 분석 요약

### 앱별 핵심 포인트

| 앱 | 강점 | 치명적 약점 | OnStep에 줄 교훈 |
|---|---|---|---|
| **Skincare Routine Diary** | 홈 위젯, 제품 효과 점수, 만료일 알림 | 체크리스트 없음, 15개 제한, 일기 기능 유료 | 위젯은 리텐션 필수 장치 |
| **Skincare Routine App** | AM/PM 분리, 레이어링 자동 정렬, ROI 계산 | 한 브랜드(DECIEM)만 DB 보유, 잔량 추적 없음 | AM/PM 분리 + 비용 추적 구조 |
| **마이루틴** | 국내 1위(150만 명), 신호등 달성률, 소셜 루틴 탐색 | 스킨케어 도메인 깊이 없음(제품 관리 없음) | 신호등 UI, 시간대별 블록 구조 |
| **Habitify** | 시간대별(아침/오후/저녁) 블록 구분 | 스킨케어 특화 없음 | 블록 단위 완료율 표시 패턴 |
| **HabitKit** | GitHub 스타일 히트맵(52주) | 스킨케어 무관 | 달력 히트맵 → LOG 화면 적용 |
| **Streaks** | 링(Ring) 완료 시각화, 위젯 생태계 최강 | 세션 개념 없음, 강제 초기화 | 링 UI → 오늘 루틴 완료 표시 |
| **(Not Boring) Habits** | 반스트릭(총 횟수 기반), 수치 심리 없음 | 내비게이션 복잡 | 세션 달성률(%) 방식 채택 |

---

### 경쟁 공백 — OnStep만이 채울 수 있는 영역

경쟁 앱 **전체**가 비어 있는 기능들:

```
✗ 제품 소진 예측 (잔량 ÷ 일일 소비량 → N일 후 소진)
✗ 세션 기반 루틴 구조 (Session #, Day 1/Day 2)
✗ 단계별 혼합 지시 (A+B 섞어, 10분 대기, C 마무리)
✗ AI 텍스트 → 루틴 자동 변환
✗ BOX(재고) ↔ SETUP(루틴) ↔ TODAY(실행) 통합 연동
```

이 5가지는 OnStep이 처음으로 한 앱에서 제공하는 차별점이다.

## 2. 사용자 기반 분석

### 타겟 페르소나

**Primary: 루틴 중급자 (25–32세 여성)**
- 스킨케어 제품 5개 이상 동시 사용 중
- 아침/저녁 루틴이 다름
- 제품 소진 타이밍을 자주 놓침
- 마이루틴 같은 앱을 쓰지만 제품 관리는 별도 메모
- 뷰지 처방전, 한글 파일에 루틴 직접 정리하는 습관

**Secondary: 루틴 고급자 (28–38세)**
- 회차(Session)별로 루틴이 바뀜 (1차, 2차 등)
- 한번에 여러 제품 구매(4병씩 대량 구매)
- 성분 간섭 신경 씀 (레티놀 + 비타민C 분리 등)
- 비용 ROI 의식 있음 ("이 세럼이 며칠치야?")

### 핵심 사용자 니즈 (우선순위 순)

```
① 오늘 어떤 순서로 뭘 발라야 하는지 바로 확인
② 제품이 언제 떨어지는지 미리 알고 싶음
③ 루틴을 까먹지 않도록 알림
④ 아침/저녁 각각 독립된 완료 체크
⑤ 루틴 수행 기록이 달력으로 보고 싶음
⑥ 제품 효과가 있는지 피부 컨디션과 연결해서 보고 싶음
```

### 사용자 행동 패턴

- 앱 오픈 시점: 화장대 앞 (아침 7–9시, 저녁 9–11시)
- 세션 길이: 체크만 할 땐 30초 이내, 기록 시 2–3분
- 가장 많이 이탈하는 시점: 루틴 구성 편집이 복잡할 때
- 리텐션 핵심 동기: "어제 뭘 했는지 보고 싶어서" → 히스토리
- 공유 행동: 루틴 결과 스크린샷 → 인스타그램/카카오톡

## 3. OnStep 적용 기능 기획

### 3-1. TODAY 화면 전면 개편

#### 현재 문제
- AM/PM 구분 없이 루틴 전체가 한 목록
- 단계별 체크 후 완료 피드백 없음
- 세션 몇 일차인지 표시 없음

#### 적용할 기능

**[A] AM/PM 블록 분리 구조** ← Habitify + Skincare Routine App
```
┌─────────────────────────┐
│  Session 2 · DAY 14     │  ← 세션 + 날짜 컨텍스트
│  오늘의 루틴             │
├─────────────────────────┤
│  🌅 아침  3/5 완료      │  ← 블록 헤더 + 진행률
│  ────────────────       │
│  ○ 물마스크 (10분 대기) │  ← 단계별 체크 + 지시
│  ✓ 델마크림             │
│  ○ 델마세럼             │
│       [아침 완료]        │  ← 블록 완료 버튼
├─────────────────────────┤
│  🌙 저녁  0/4 완료      │
│  ────────────────       │
│  ○ 클렌저               │
│  (잠금 상태 — 저녁 시작 │  ← 시간대 전 흐리게
│   후 활성화)            │
└─────────────────────────┘
```

**[B] 루틴 신호등** ← 마이루틴
- 오늘 달성률: 0–29% 🔴, 30–69% 🟡, 70–100% 🟢
- 상단 or 탭바 아이콘에 색상 닷(dot) 표시

**[C] 완료 마이크로인터랙션** ← (Not Boring) Habits + Streaks
- 단계 체크 시: 빠른 체크 애니메이션 + 햅틱 (light)
- 아침/저녁 블록 완료 시: 코랄 핑크(#E94F6B) 플래시 + 햅틱 (medium)
- 하루 전체 완료(AM+PM) 시: 전체화면 파티클 + "Session 2, Day 14 완료!" 메시지

**[D] 대기 타이머** ← Skincare Routine App
- 단계에 waitMinutes가 설정된 경우 타이머 자동 시작
- 화면 상단 미니 타이머 배너 ("물마스크 — 7:23 남음")
- 타이머 완료 시 알림

**[E] 세션 달성률 표시** ← (Not Boring) Habits 방식
```
Session 2 진행: ████████░░  14/30일  46%
이번 세션 달성률: 아침 92% · 저녁 78%
```
- 스트릭(연속일) 대신 세션 내 달성률(%)로 표시
- 심리적 부담 없이 꾸준함 시각화

### 3-2. LOG 화면 재설계

#### 현재 문제
- 단순 날짜 리스트
- AM/PM 구분 없음
- 장기 패턴이 안 보임

#### 적용할 기능

**[A] AM/PM 이중 도트 달력** ← HabitKit 히트맵 변형
```
     6월
 월  화  수  목  금  토  일
  ·  ●  ●  ●  ○  ●  ○    ← 위: 아침, 아래: 저녁
  ·  ●  ○  ●  ●  ●  ●

● 아침+저녁 완료   ○ 아침만   · 미완료
```
- 각 날짜 셀: 상단 점(아침) + 하단 점(저녁)
- 색상: 완료=#E94F6B, 아침만=rgba(233,79,107,.4), 미완료=#222

**[B] 세션 구간 표시**
- 달력에 세션 1, 세션 2 구간 배경색으로 구분
- 세션 교체 지점에 구분선 + 라벨

**[C] 피부 컨디션 기록** ← Skincare Routine Diary + 마이루틴
- 루틴 완료 후: "오늘 피부 컨디션?" 1–5 이모지 선택 (3초 내)
- 선택적 메모 입력
- LOG 화면 달력 셀에 컨디션 색상 overlay

**[D] 연속 완료 기록 (세션 내)**
- 세션 내 연속 완료일 표시 (오늘 기준 현재 스트릭)
- 세션 종료 시 "Session 2 총 달성률 89%" 요약 카드 생성

---

### 3-3. BOX 화면 개선

**[A] 소진 예측 표시** (이미 계산 로직 구현됨 → 카드에 노출)
```
┌────────────────────────┐
│ 델마세럼               │
│ ████████░░  잔량 62%   │
│ ⏱ 약 37일 후 소진     │ ← 이미 계산, 카드에 추가
│ 📅 7월 8일 예상 소진  │
└────────────────────────┘
```

**[B] 재구매 타이밍 알림**
- 소진까지 N일 이하(기본 14일) → "재구매 준비하세요" 뱃지
- 유통기한 < 소진 예상일 → "유통기한 먼저 끝납니다" 경고

**[C] ROI/비용 추적** ← Skincare Routine App
- 제품 가격 + 총 용량 + 1회 사용량 → 1회당 비용 계산
- 예: "델마세럼 — 1회 약 ₩290"
- 카드 하단에 작게 표시

---

### 3-4. SETUP 화면 개선

**[A] 레이어링 순서 가이드** ← Skincare Routine App
```
권장 레이어링 순서:
1. 클렌저
2. 토너/에센스
3. 수용성 세럼
4. 유성 세럼
5. 오일
6. 크림
7. 선크림 (아침만)
```
- 제품 추가 시 카테고리 기반 자동 순서 제안
- 사용자가 수동 변경 가능

**[B] 성분 주의 경고** ← Skincare Routine App (간소화)
- 특정 조합 감지: 레티놀 + 비타민C → 🟡 주의 표시
- 레티놀 + AHA → 🔴 함께 사용 주의
- 첫 단계: 수동 성분 태그 기반으로만 (AI 없이)

---

### 3-5. 신규 기능: 루틴 공유

**← Skincare Routine App + 마이루틴**

- "오늘 루틴 공유" 버튼 → 이미지 생성
- OnStep 브랜딩 + 세션 정보 + AM/PM 단계 리스트
- 인스타그램 스토리 최적화 (9:16 비율)

---

### 3-6. 신규 기능: 홈 위젯

**← Skincare Routine Diary (강력히 추천) + 마이루틴 + Streaks**

위젯 콘텐츠:
```
┌──────────────────┐
│  OnStep          │
│  Session 2 · D14 │
│  ─────────────── │
│  🌅 아침  ████░  │ 3/5
│  🌙 저녁  ░░░░░  │ 시작 전
│  [아침 시작 →]   │
└──────────────────┘
```

- 소형(2×2): 오늘 달성률 링 + 탭하면 앱 오픈
- 중형(4×2): AM/PM 블록 진행률 + 현재 다음 단계
- 잠금화면 위젯: 달성률 숫자 + 색상 닷

## 4. 디자인 적용 방안

### 4-1. TODAY 화면 컴포넌트 디자인

**블록 헤더 (AM/PM 구분)**
```
배경: rgba(255,255,255,.04)
코너: 16px
패딩: 16px
헤더 색상: 아침=#FFB300, 저녁=#7B68EE
진행 바: 얇은 2px 라인, 완료분=#E94F6B
```

**스텝 카드**
```
높이: 56px
레이아웃: [체크박스] [제품명 + 지시사항] [타이머/뱃지]
체크박스: 24px 원형, 미완료=테두리만, 완료=코랄 핑크 채움
완료 시: 제품명 취소선 + 투명도 50%
```

**완료 애니메이션 스펙**
```
단계 체크: 체크마크 scale 0→1 (200ms ease-out) + light haptic
블록 완료: 헤더 배경 코랄 플래시 (400ms) + medium haptic
전체 완료: 파티클 burst 1.2초 + "완료!" 팝업 + heavy haptic
```

### 4-2. LOG 달력 디자인

**날짜 셀 구조**
```
┌───┐
│ 5 │  ← 날짜
│● ○│  ← 아침(좌), 저녁(우) 도트 6px
└───┘

색상 매핑:
완료 ●: #E94F6B
미완료 ○: rgba(255,255,255,.2)
없음  ·: 빈 공간
```

**세션 구간 배경**
```
Session 1 배경: rgba(197,255,0,.04)
Session 2 배경: rgba(233,79,107,.04)
구분선: 1px dashed rgba(255,255,255,.2)
```

### 4-3. 신호등 UI

**위치**: TODAY 탭 아이콘 우측 상단 닷
```
0–29%:   #D93025 (빨강)
30–69%:  #FFB300 (노랑)
70–100%: #C5FF00 (라임 그린)
```

**헤더 달성률 바**
```
위치: TODAY 화면 최상단
높이: 3px
배경: rgba(255,255,255,.1)
채움: 달성률에 따른 신호등 색상
전환: transition 0.5s ease
```

### 4-4. BOX 카드 소진 예측 디자인

**소진 예측 행 추가 (카드 하단)**
```
기존:
[██████░░] 62%

추가:
[██████░░] 62%
⏱ 약 37일 후 소진 · 1회 ₩290
```

**임박 경고 상태**
```
14일 이내: 텍스트 색상 #FFB300
7일 이내: 텍스트 색상 #D93025 + LOW 뱃지
```

### 4-5. 타이포그래피 & 색상 추가 정의

```css
/* 세션 컨텍스트 텍스트 */
font: 11px DM Sans, letter-spacing 0.08em, uppercase
color: rgba(255,255,255,.4)

/* 아침 색상 */
--morning: #FFB347  /* 따뜻한 오렌지 */

/* 저녁 색상 */
--evening: #8B7CF6  /* 라벤더 퍼플 */

/* 완료 상태 */
--done: #E94F6B     /* 기존 코랄 핑크 — 완료의 의미 부여 */

/* 신호등 */
--signal-red:    #D93025
--signal-yellow: #FFB300
--signal-green:  #C5FF00
```

## 5. 실행 단계 (Stage별 구현 계획)

### Stage A — TODAY 화면 AM/PM 분리 (2주)

**목표**: 가장 임팩트 큰 기능부터. 매일 쓰는 화면이므로 최우선.

**구현 항목:**
- [ ] `app/today/page.tsx` 리팩터링: AM/PM 블록 컴포넌트 분리
- [ ] 각 블록 헤더: 아이콘 + 진행률 카운트 + 색상 바
- [ ] 스텝 카드: 체크박스 + 제품명 + 지시사항 1줄
- [ ] 블록 완료 버튼 ("아침 완료 ✓")
- [ ] 단계 체크 애니메이션 (CSS transition)
- [ ] 블록 완료 색상 플래시

**데이터 연결:**
```typescript
// 오늘 날짜 기준 활성 루틴 조회
// session.days[오늘 dayNumber].steps 에서 AM/PM 분리
// usageLogs 에 완료 기록 저장
```

**완료 기준**: 아침/저녁 각각 독립 체크리스트가 작동하면 완료.

---

### Stage B — 세션 컨텍스트 + 달성률 (1주)

**목표**: "지금 몇 회차 며칠차인지" 한눈에 보이게.

**구현 항목:**
- [ ] TODAY 상단 세션 정보 헤더: "Session 2 · DAY 14"
- [ ] 세션 내 달성률 계산: 완료일수 / 총 세션일수

## 7. 경쟁 앱 대비 OnStep 포지셔닝

```
                 스킨케어 도메인 깊이
           낮음 ◄────────────────► 높음
           │                       │
높음 │ 마이루틴      │             │ OnStep (목표)
     │ (1.5M명,     │             │ 세션관리+소진예측
     │  도메인 없음) │             │ +루틴실행+재고
     │               │             │
     │               │ Skincare    │
낮음 │               │ Routine App │
     │               │ (레이어링)  │
     └───────────────┴─────────────┘
```

**OnStep의 전략적 포지션**: 마이루틴의 사용자 규모를 목표로 하되, 스킨케어 도메인에서 압도적인 깊이로 차별화. 마이루틴 사용자 중 스킨케어를 진지하게 관리하는 층이 타겟.

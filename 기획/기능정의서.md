## 2. Today 화면

### 2-1. #OOTD (오늘의 룩)

#### 테마 표시
- 오늘 일정(`scheduledDates`)에 포함된 룩 테마를 자동 선택해 히어로 이미지로 표시
- 히어로 이미지: 테마 `baselineImg`
- 연결 제품: BOX 이미지 우선, 없으면 아이콘
- 우측 상단 ⚙ → `log.html?tab=lookbook&id=THEME_ID` (해당 컨텐츠 상세 직접 오픈)
- 헤더 **Edit →** → `log.html?tab=lookbook` (룩북 라이브러리 리스트)

#### 오늘의 룩 기록
- **Record your final look** 카드 탭 → 기록 시트 오픈
- 기록 시트 구성
  1. **룩 테마 선택** — 라이브러리에 등록된 룩 테마 목록, 오늘 일정 테마 자동 선택
  2. **사진** (선택) — 탭하여 카메라/갤러리에서 첨부, 480px 압축
  3. **메모** (선택) — 커스텀 메모; 비워두면 선택한 테마 이름이 자동 기록
- **저장** → `onstep_looks` 로컬 저장 → Firestore `extra/data.looks` 동기화 → **Log 아카이브 자동 반영**
- **삭제** — 기록 카드 ✕ 버튼 또는 시트 내 "기록 삭제" 버튼 (확인 다이얼로그 후 실행)

#### 자정 초기화 규칙
- 오늘의 룩 기록은 날짜 키(`YYYY-MM-DD`)로 관리됨
- **자정(00:00)**이 지나면 다음 날 기록이 없는 상태로 자동 초기화 표시
- 구현 방식
  - `setTimeout` : 자정까지 남은 시간 계산 → 자정 직후 `renderTodayOOTD()` 재실행 → 재귀로 다음 자정 예약
  - `visibilitychange` : 앱으로 돌아왔을 때 날짜 키가 바뀌어 있으면 즉시 재렌더
- **기존 기록은 삭제되지 않음** — `onstep_looks` 내 과거 날짜 데이터는 유지, 오직 Today 표시만 새 날짜 기준으로 전환

## 3. Log 화면

### 3-1. Library 탭

| 구분 | 필터 | 데이터 키 |
|---|---|---|
| Makeup | `pill-makeup` | `onstep_makeup_contents` |
| Lookbook | `pill-lookbook` | `onstep_look_contents` |

#### 컨텐츠 카드 구조
1. **히어로 이미지** — 전체 폭 220px, `baselineImg`
2. **카드 바디** — 이름, 설명, 자동/일정/횟수 배지, 일정 추가 버튼
3. **제품 스트립** — BOX 이미지 포함, 3개 반 노출 후 좌우 스크롤

#### 일정 등록
- `+ 일정` 버튼 → 날짜 시트 → `scheduledDates[]`에 추가
- AUTO 모드 활성화 시 일정 기반 자동 노출

#### 컨텐츠 상세 패널
- 좌 → 우 슬라이드, 편집/삭제/일정/BOX 제품 연결 기능 포함
- URL 딥링크 지원: `log.html?tab=lookbook&id=123` → 해당 컨텐츠 상세 자동 오픈

- 최신 날짜 순 내림차순

## 4. 데이터 흐름

### 오늘의 룩 저장 → 아카이브 반영 흐름

```
[Today] saveOOTDRecord()
  └─ localStorage: onstep_looks 업데이트
  └─ Firestore: users/{uid}/extra/data.looks 동기화 (saveLogsToCloud)
        ↓
[Log] renderArchiveMain()
  └─ readHistory() ← onstep_looks 읽기
  └─ lib-card 형식으로 렌더
```

### 자정 초기화 흐름

```
페이지 로드 → _scheduleMidnightReset()
  └─ setTimeout(자정까지 ms)
        └─ 자정 → renderTodayOOTD() 재실행 → 새 날짜 기준으로 "기록 없음" 표시
        └─ _scheduleMidnightReset() 재귀 호출 (다음 자정 예약)

visibilitychange → getTodayKey() !== _currentDateKey?
  └─ YES → renderTodayOOTD() + renderTodayMakeup() 즉시 갱신
```

## 6. 핵심 설계 원칙

1. **날짜 키 기반 격리** — 오늘 기록은 `YYYY-MM-DD` 키로 분리, 과거 데이터 불변
2. **사진 우선순위** — 사용자 첨부 사진 > 테마 기본 이미지 > 플레이스홀더
3. **Firestore = 원본, localStorage = 캐시** — 모든 저장은 두 곳 동시 수행
4. **실시간 동기화** — `BroadcastChannel`로 탭 간 즉시 반영
